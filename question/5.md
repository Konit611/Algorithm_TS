問題 2: チケットの状態遷移管理 (State Machine)
サポートチケットのように、決められた順序でしかステータスを変更できないロジックを実装してください。

要件
1. 型定義：
- TicketStatus: 'open' | 'in_progress' | 'closed'
- Ticket: { id: string; status: TicketStatus; title: string; }

2. 関数 updateTicketStatus(ticket: Ticket, nextStatus: TicketStatus): Ticket を作成すること。

3. 遷移ルール：
- open → in_progress は 許可
- in_progress → closed は 許可
- それ以外の遷移（例: open → closed）や、一度 closed になった後の変更は 不可。
- 不正な遷移の場合は、エラーを投げるか、元の ticket をそのまま返してください。

4. 期待値
```typescript
/**
 * [現在の状態, 次の状態, 期待される最終状態, 説明]
 */
const ticketTestData = [
  ["open",        "in_progress", "in_progress", "正常な遷移"],
  ["in_progress", "closed",      "closed",      "正常な終了"],
  ["open",        "closed",      "open",        "スキップ不可"],
  ["closed",      "open",        "closed",      "終了後の再開不可"],
  ["in_progress", "open",        "in_progress", "後戻り不可"],
  ["open",        "open",        "open",        "同一ステータス維持"],
] as const;
```
